# Hooroo App 视觉体感内容 MVP

## 📋 项目概述

Hooroo App 是一款体感游戏和健身应用。本项目为新版本的「视觉体感内容」MVP设计，目标是引入视觉AI动作识别技术，让用户通过手机摄像头进行体感游戏，并支持投屏到TV。

## 🎯 项目目标

### 核心目标
1. **内容目标**: 提供至少9个高质量视觉体感内容（舞蹈、健身、游戏各3个）
2. **转化目标**: 新用户进入体感内容的漏斗流失率 ≤ 30%，单次游戏时长 ≥ 5分钟
3. **留存目标**: 新用户次日留存率达到30%

### 功能需求
- ✅ 新手引导：登录注册引导、体感游戏操作引导
- ✅ 信息架构优化：新增视觉体感内容区域，保留手表体感内容
- ✅ 视觉体感游戏：舞动时刻、健身系列、热门游戏改造
- ✅ 投屏功能：引导用户投屏、自动检测设备
- ✅ 用户反馈：反馈入口、游戏后评价系统

## 📱 双体感模式设计

### 🆕 视觉体感（新功能）
- **突出展示**: 首页顶部大卡片推荐，配有"新功能"标签
- **彩色设计**: 使用彩色渐变卡片，视觉冲击力强
- **功能说明**: 强调"无需手表，摄像头即可"的优势
- **内容丰富**: 舞蹈、健身、游戏三大类，共10个内容
- **默认展示**: 体感页面默认显示视觉体感内容

### ⌚ 手表体感（经典模式）
- **保留展示**: 首页下方区域展示，标注"经典模式"
- **灰色调设计**: 使用较为低调的灰色调，不抢夺视觉焦点
- **连接提示**: 明确提示"需要连接智能手表"
- **成熟内容**: 跑步、力量、有氧三大类，共10个内容
- **筛选访问**: 通过筛选按钮可以单独查看手表体感

## 📋 信息架构设计

### 设计原则
1. **突出新功能**: 视觉体感作为主推功能，放在显眼位置
2. **保留原功能**: 手表体感作为成熟功能，继续提供给用户
3. **清晰区分**: 用不同的视觉设计区分两种体感方式
4. **引导体验**: 优先引导用户尝试新的视觉体感功能

### 具体调整
- **首页**: 新增视觉体感推荐区域，保留手表体感展示
- **体感页**: 增加筛选功能，支持视觉/手表/全部三种模式
- **视觉标识**: 新功能用彩色卡片，经典功能用灰色调
- **引导策略**: 默认展示视觉体感，手表体感需要连接提示

## 🚀 用户流程

### 新用户流程
1. **新手引导** → 了解视觉体感概念和特色功能
2. **首页浏览** → 优先看到视觉体感推荐，了解手表体感选项
3. **选择体感方式** → 根据设备情况选择视觉或手表体感
4. **游戏体验** → 进入游戏，完成体感挑战
5. **结果反馈** → 查看成绩，提供反馈，分享成就

### 老用户流程
1. **快速访问** → 通过筛选功能快速找到熟悉的手表体感内容
2. **新功能尝试** → 被首页推荐吸引，尝试视觉体感
3. **混合使用** → 根据场景选择不同的体感方式

## 💡 用户引导策略

通过视觉层次和交互设计，优先引导用户体验新的视觉体感功能，同时保留原有手表体感的完整功能：

- **新用户**: 被新功能的彩色设计和"立即体验"按钮吸引
- **老用户**: 通过筛选功能快速找到熟悉的手表体感内容
- **渐进式引导**: 不强制用户选择，提供清晰的选项和说明
